# Incorrect Use of TransactionTypeAxis with Transaction Market Method Members
Rule ID: DQC.US.0241.10942  
Status: Final  
Release date: June 29, 2026  
Effective date: September 1, 2026   

## Rule Function

**Purpose:**

The purpose of Rule DQC.US.0241 is to ensure that filers do not use `TransactionTypeAxis` with market method members (`OverTheCounterMember`, `ExchangeTradedMember`, or `ExchangeClearedMember`). The 2026 US GAAP taxonomy introduced `TransactionMarketMethodAxis` specifically for disaggregating transactions by their market or clearing method. `TransactionTypeAxis` is intended to categorize the type of transaction (e.g., Financial Assets Sold under Agreement to Repurchase, Securities Loaned, Sale and Total Return Swap) — not the venue or method by which the transaction is executed or cleared.

**Conditions:**

Assertion 10942 fires when:

- A fact is tagged with `TransactionTypeAxis` using any of the following members:
  - `OverTheCounterMember`
  - `ExchangeTradedMember`
  - `ExchangeClearedMember`

## Problem Solved by the Rule

Prior to the 2026 taxonomy, preparers had no dedicated axis for identifying whether a transaction was over-the-counter, exchange-traded, or exchange-cleared. Some filers applied these market method members to `TransactionTypeAxis` as a workaround. The 2026 taxonomy addresses this gap by introducing `TransactionMarketMethodAxis`, which is the correct axis for these members. Continuing to use these members on `TransactionTypeAxis` creates a semantic mismatch — the axis implies a transaction type distinction, but the members reflect an execution venue distinction. This misclassification can distort how data consumers interpret financial instrument disclosures.

## Example Rule Message

The filer has reported a fact with the **TransactionTypeAxis** that is tagged with the concept **FinancialAssetsSoldUnderAgreementsToRepurchaseGrossIncludingNotSubjectToMasterNettingArrangement** with a value of 350000000. The `TransactionTypeAxis` should not be used with these transaction market method items. The filer should use the `TransactionMarketMethodAxis` for disaggregation of these transaction market method items.

## Technical Details

Assertion 10942 retrieves any fact tagged with `TransactionTypeAxis` where the member value is one of `OverTheCounterMember`, `ExchangeTradedMember`, or `ExchangeClearedMember`. If any such fact is found, the assertion fires.

Filers should:
- Replace `TransactionTypeAxis` with `TransactionMarketMethodAxis` when disaggregating by over-the-counter, exchange-traded, or exchange-cleared method.
- Reserve `TransactionTypeAxis` for classifying the nature of the transaction itself (e.g., repurchase agreements, securities lending, total return swaps).

The `TransactionMarketMethodAxis` and its associated members are new in the 2026 US GAAP taxonomy.
